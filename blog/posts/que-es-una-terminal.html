¿Qué es una terminal de comandos?

Armando Alamilla
Mayo 31, 2020
4 comentarios

Las personas y las computadoras no hablan el mismo lenguaje. Para que exista una comunicación entre el usuario y la computadora se necesita de un intermediario que pueda entender nuestro lenguaje y lo traduzca a instrucciones que la computadora pueda procesar. Debemos tener esto en mente porque de eso se trata la terminal. Vamos a revisar algunos conceptos.

¿Qué es una terminal?

La terminal es un programa que funciona como intermediario entre el usuario y la computadora. Recibe instrucciones conocidas como “comandos” que son convertidos a código que la computadora puede procesar y ejecutar para dar un resultado.

La terminal trabaja con una interfaz de línea de comandos (CLI), esto quiere decir que el usuario interactúa con la computadora mediante texto introducido desde el teclado. A diferencia de una interfaz gráfica (GUI), la interfaz de linea de comandos (CLI) nos da tres principales ventajas:

  • Velocidad
  • Eficiencia
  • Automatización de tareas

Partes que componen a la terminal

Prompt: es el conjunto de caracteres que se muestran en la parte izquierda de la terminal para indicar que ahí se pueden ingresar comandos. El prompt puede ser configurado por lo que su visualización dependerá del sistema pero en sistemas unix y linux por defecto mostrará información importante como el nombre de usuario y el nombre de la máquina de la siguiente forma: usuario@máquina:~$

Cursor: es un indicador que muestra la posición donde será colocado el texto que se ingrese a la terminal. El cursor puede estar representado gráficamente dentro de la terminal como un guión bajo, una barra vertical o un rectángulo sólido y puede estar fijo o parpadeando.

Estructura de los comandos

Los comando tienen una estructura básica de tres partes: el nombre del comando, modificadores (conocidos también como banderas) y parámetros (conocidos también como argumentos).

Nombre: Como primer requisito debe ponerse el nombre del comando. Es esencial porque así el sistema sabe qué instrucción ejecutar.

Modificadores: (Conocidos también como banderas) Son opciones que modifican el compotamiento de un comando y se escriben después de un "-" o en algunos casos "--". Ejemplo: -flag

Parámetros: (Conocidos también como argumentos) son valores que requieren algunos comandos para ejecutar las instrucciones.

Ejemplos de comandos

date (Muestra la fecha y hora)
Es un comando simple porque no requiere modificadores ni parámetros.

ls -l (Lista los elementos que contiene una carpeta)
Al agregarle el modificador "-l" los elementos se muestran en formato de listado largo.

echo "Hola mundo" (Imprime un mensaje en la pantalla)
Se le debe pasar como parámetro el mensaje que se va a imprimir en la pantalla. En este ejemplo el mensaje es "Hola mundo"

rm -rf fotos/ (Elimina una carpeta)
El comando "rm" por defecto solo elimina archivos. Para poder eliminar carpetas se tienen que agregar los modificadores "-rf" y como argumento el nombre de la carpeta (directorio) que se quiere eliminar.

Consejos y atajos para la terminal

Presionar la tecla [Tab] para autocomplementar el comando que vas a escribir, en caso de haber múltiples opciones se tendrá que presionar dos veces la tecla [Tab].

Presionar la flecha de arriba [↑] en el teclado para visualizar en el prompt los comandos que se han ingresado con anterioridad.

Ingresar el comando “history” para ver el historial de comandos que han sido ingresados en la terminal. Te muestra el id del comando y su nombre, si quisieras volver a ingresar algún comando puedes hacerlo escribiendo “!” seguido del id del comando. Ejemplo: !498 esto ejecutará el comando que en el historial tenga el id 498

Combinación de teclas CTRL + L para borrar la pantalla.
Combinación de teclas CTRL + R para buscar comandos usados anteriormente.
